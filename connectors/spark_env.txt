📄 دليل ربط VS Code ببيئة تشغيل Spark Notebook (Jupyter Server)
================================================================================

لربط محرّر VS Code مباشرة بالخادم (Jupyter Server) الذي يعمل داخل حاوية 
Docker، حتى تتمكن من تشغيل الخلايا (Cells) البرمجية داخل VS Code مباشرة، 
اتبع الخطوات التالية بدقة:

--------------------------------------------------------------------------------
1️⃣ تحديد نوع الخادم (Select Server Type)
--------------------------------------------------------------------------------
* افتح لوحة اختيار الـ Kernel في VS Code.
* اضغط على خيار: "Existing Jupyter Server..." (خادم Jupyter الحالي).

--------------------------------------------------------------------------------
2️⃣ إدخال رابط الاتصال (Enter Connection URL)
--------------------------------------------------------------------------------
* سيطلب منك VS Code إدخال رابط الخادم. قم بكتابة أو لصق الرابط التالي:

  URL: http://127.0.0.1:8889/?token=admin

* ثم اضغط على زر [Enter].

--------------------------------------------------------------------------------
3️⃣ تسمية الاتصال (Name the Connection)
--------------------------------------------------------------------------------
* سيطلب منك VS Code تسمية هذا الاتصال لسهولة التعرف عليه لاحقاً.
* اكتب اسماً معبراً، مثل:
  
  Name: Docker Spark Notebook

* ثم اضغط على زر [Enter].

--------------------------------------------------------------------------------
4️⃣ اختيار بيئة التشغيل (Select Kernel)
--------------------------------------------------------------------------------
* ستظهر لك قائمة ببيئات التشغيل (Kernels) المتاحة داخل الحاوية.
* اختر البيئة الافتراضية التالية:

  Kernel: Python 3 (ipykernel)

--------------------------------------------------------------------------------
🚀 النتيجة النهائية
--------------------------------------------------------------------------------
الآن سيقوم VS Code بتشغيل أي ملف Notebook (.ipynb) مباشرةً داخل بيئة الحاوية 
المعزولة التي تحتوي على كافة المكتبات مهيأة مسبقاً (PySpark, ClickHouse, MinIO).
